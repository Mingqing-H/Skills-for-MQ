# 模板规范与扩库流程

每个模板放在 `references/template-<id>.md`，并由 `SKILL.md` 直接链接。不要建立多层目录。

## 必需 frontmatter

```yaml
---
id: museum-exhibition
name: 博物馆展览海报
family: culture
version: 1.0.0
---
```

`id` 使用小写字母、数字和连字符，并与文件名一致。`family` 使用稳定分类：构图骨架统一使用 `composition`，题材模块使用 `concept`、`commercial`、`editorial`、`event`、`culture` 或 `brand`。`version` 使用语义版本。

## 必需章节

1. `## 适用场景`
2. `## 必填变量`
3. `## 设计机制`
4. `## 提示词骨架`
5. `## 质量检查`

变量统一写成 `{{变量名}}`。骨架必须包含版式、主体、图文、色彩/光材质和避免项。

## 从案例抽象模板

只沉淀阅读层级，网格、裁切、尺度和运动方向，图像与文字的空间关系，色彩角色与材质对比，系列化不变量，以及常见失败限制。

删除或变量化真实品牌名、logo、人物身份、赛事标志、原案例文案/日期/价格/产品、受保护角色和高度独特的复刻要求。不要仅凭作者名或风格名触发模仿。保留来源记录时，只写链接、日期和借鉴机制，不复制整条长提示词。

## 两条扩展轴

- 构图轴回答“视觉重量、锚点、路径和文字区如何组织”。
- 场景轴回答“画什么、用什么材质、怎样避免该题材的常见失败”。

新增前先判断它属于哪一轴。不得用产品、餐饮、时尚等题材名称代替构图方法。

## 决定新增还是合并

仅当阅读结构显著不同、出现新的稳定图文机制、目标媒介有不同信息纪律，或新模板至少覆盖三个合理场景时新增。否则合并为现有模板的可选变化。

## 修改清单

1. 用 `scripts/new_template.py` 创建模板。
2. 填写变量、机制、骨架和质量检查。
3. 在 `SKILL.md` 路由表添加直接链接。
4. 执行 `python scripts/validate_templates.py`。
5. 用真实简报试填，确认没有空变量、冲突或来源依赖。
