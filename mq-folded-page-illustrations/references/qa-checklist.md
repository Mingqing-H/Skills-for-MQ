# QA Checklist

## 必过项

- 16:9 横版，纯白背景，至少约 35% 留白。
- 折页员与参考图一致：米白折纸身体、左上折角、右侧内折、黑色窄视窗、右上回形针、细肢体。
- 胸口 MQ 朱印存在；`M` 与 `Q` 近看可辨，远看仍像小印章。
- 折页员承担核心动作，不只是站在旁边。
- 主轮廓干涩有轻重，结构线更细；没有统一粗描边或光滑矢量感。
- 钴蓝只用于工程辅助线、反馈或第二层状态。
- 朱红主要用于 MQ 印章，最多再有一处关键警告或结论。
- 中文是窄体硬笔批注；3-6 处、每处 2-6 字、可读。
- 一张图只讲一个核心结构，没有复刻旧案例。

## 失败信号

- 角色变成信封、购物袋、机器人、动物或黑色实心怪物。
- 出现圆眼、嘴、眉毛、服装；回形针、窄视窗或 MQ 朱印丢失。
- MQ 看成随机迷宫纹，或印章大得像品牌 Logo。
- 线条过粗、等宽、光滑，或出现大面积灰色阴影。
- 中文像圆体、手账体、泡泡字、毛笔字或印刷字体。
- 左上角有“流程图 / 系统架构 / 路线图 / 常见坑”等类型标题。
- 画面像 PPT、课程页、正式流程图或科技 UI。
- 元素、箭头、节点或文字过多。
- 背景出现纸纹、米色、渐变、阴影或噪点。
- 直接沿用旧版小黑案例的构图或角色动作。

## 迭代方法

- 角色漂移：用角色参考图进行局部身份修复，重申固定折角、视窗、回形针和 MQ 印章。
- 印章不清：只编辑印章，明确 `M` 上、`Q` 下，保留方框与断墨。
- 太普通：让折页员用折、夹、压、展、量等动作改变结构。
- 太复杂：删到 1 个主物件、1 个动作、3-5 个短标注。
- 太 PPT：移除标题、卡片、整齐网格和多余箭头，改成一个手绘场景。
- 文字不对：只编辑文字，指定窄体硬笔与原词原位，不重新生成整图。

## 交付判断

合格图片应先让读者看到一张干净的工程草图，再在一秒内理解折页员完成的动作。视觉个性来自角色、线条和批注系统，而不是靠复杂装饰。
